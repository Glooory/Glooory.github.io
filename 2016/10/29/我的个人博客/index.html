搭建缘由

以前总觉得拥有一个自己专属的域名好像很酷的样子，今天我也拥有了自己的专属域名了，

让我先好好享受下，然而好像除了精神上的慰藉，并没有什么用。

其实老早以前就听说过个人博客了，最早听说这个词还是初中毕业那会吧，那时候写什么同

学录，上面有个东西叫部落格叫我填，当时书读得少真不知道那是什么玩意，后来才知道那玩

意就是Blog的音译，好吧，真的是书读得少不知道。真正近距离、频繁接触到博客还是开始

学习 Android 后，因为大家有什么技术文章都会放到自己的博客中，方便自己也造福于他人。

其实我几个月前就按照教程搭建了 GitHub Pages 了，只不过上面除了一张图什么都没有，也

没有申请和绑定独立域名，我还跟一位小伙伴炫耀来着，现在想想还真是天真幼稚。由于以前

写的项目就只把代码放 GitHub上了，什么介绍也没有，因为当时也不知道 MarkDown 语法，

所以项目介绍都是空白，现在想重新整理一下，添上项目介绍顺便贴点东西在自己的博客上，

所以就开始着手搭建自己的博客了。

搭建过程

网上这类教程太多了，结合起来看还是挺好弄好的

整个过程画了我两天时间，期间还遇到了不少难题，最搞笑的是我这个月代理软件的流量之前

被我不小心作死用来下载视频用了太多了，就是这货：

流量被我糟蹋完了，而且我后来还找到了这场演唱会的 1080p 版，真的是浪费我流量，这下谷

歌用不了了，只能用国内的搜索引擎了，以前每个月的流量我一般一半都用不到的，偏偏这个

时候给我整这么一出，只能忍一忍等下个月满血复活了。从昨天早上开始按照教程开始折腾，

在选什么主题时费了相当多时间，本来最初准备使用 Next 主题的，但由于太多人用以及有些细

微的地方不合我的的胃口，所以就有点犹豫不决，就在这时候看见了这个主题 light_cn 还蛮符

合我的整体胃口的，够简洁，于是就敲定使用它了，配置过程中遇到了些小麻烦，其实 作者 的

博客中都找得到相应的解决办法，但由于自己并没有注意到导致做了许多无用功，还好结果挺好

的，基本达到了自己满意的样子，后续有机会在逐渐完善。

在配置过程中还去申请了下域名，运气比较好，我现在使用的 ID 相关的域名都还有，挺高兴的，

目前买了一年先试一下。我感觉记录生活琐事应该会比技术文章多一点，因为我现在还在为成为

一个菜鸟 Android Developer 努力，按照我现在的状况，其实说实话我也不知道自己还能坚持到

什么时候，总之还是加油吧。

总结

感觉从高考之后就没怎么写过东西了，这篇文章写的什么鬼也不知道，就是想到哪到写到哪。

总之以后会在这耕种自己的一亩三分地，记录些我自己的想法和事情。

This ship is ready to sail！
