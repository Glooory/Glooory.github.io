更改传输协议

今天使用 Git 工具向我的 pages repo push 时每次都要输入账户名和密码，后来在网上查了

一下，原来是我以前关联远程库的时候使用了 https 的方式，所以每次都要用 https 的方式

push 到远程库，于是就找解决办法重新设置成了 ssh 的方式。

首先查看当前使用的传输协议：

git remote -v

如果输出 origin https://github.com/username/repository.git 说明使用的是 https 方式，

如果输出 origin git@github.com:username/repository.git 则表明使用的是 ssh 方式。

重置成 ssh 的方法:

就是先取关再重新以 ssh 的方式关联远程库。

刚看见了还有另一种方法，更简单：

git remote set-url origin git@github.com:username/repository.git

There is no tracking information for the current branch

由于我上面有取关远程库再重新以另一种方式关联的操作，当我 pull 的时候就报了如下错误：

原因是本地分支和远程分支没有建立联系，不知道是不是我刚才更改传输协议的时候造成的，

使用git remote -vv 查看本地分支和远程分支的关联关系，输出如下：

master 8129bac commit_information

执行以下命令进行本地和远程分支间的关联：

git branch --set-upstream-to=origin/远程分支的名字 本地分支的名字

再查看关联关系输出如下：

master 8129bac [origin/master] commit_information

对比前后两次输出就能发现为什么会出现该问题了，现在可以愉快的 pull 和 push 了。
